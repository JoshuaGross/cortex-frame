PHP-Frame Template

Goals

  • HTML markup, like PHP code, should not be duplicated. INCLUDE directives should allow for modular templates (irrelevant with WYSIWYG packages like Dreamweaver).
  • The template engine needs to be able to play nice with WYSIWYGs like Dreamweaver.
  • The framework should make it easy to transfer data between PHP and the template engine. phpBB and other template engines require too much pointless re-structuring of data.
  • No PHP code in the template. Nothing resembling it. Period.
  • Template markup HTML can only access variables sent to the engine.
  • Template markup can loop through arrays, with unlimited nesting.
  • Lightweight.
  • Simplicity.

Suggested practices

  • Consistent variable naming schemes. Everything is lower-case. Prefix all URLs with "u_", images with "u_img_", thumbnails with "u_thumb".
  • Turn on server-side Template caching for any large project.
  • Use $template->set_var_safe instead of $template->set_var when possible. set_var_safe runs htmlspecialchars on the input.
  • If Dreamweaver or another WYSIWYG tool is being used, place template HTML files in the document root, named tpl_xxxx.html. That way, relative URL paths can be used.
